# Component Guidelines

> Component conventions for this project.

---

## Overview

There are currently no frontend components in this repository. Do not invent component, prop, styling, or accessibility conventions for non-existent UI code.

If a future task introduces a web UI, create or update this file from real implementation choices before adding more components.

---

## Component Structure

Not applicable today.

Future component specs should document:

- component file naming

## Props Conventions

Not applicable today. There are no component props.

---

## Styling Patterns

Not applicable today. There is no CSS, Tailwind, CSS modules, styled-components, or design token system.

---

## Accessibility

Not applicable today because there is no browser UI. CLI accessibility is handled through predictable stdout/stderr behavior and machine-readable JSON/Markdown output; see backend quality and error-handling specs.

---

## Common Mistakes

Do not treat npm wrapper files as frontend components.

Do not add placeholder React/Vue/etc. guidance without real project code. Future AI agents should follow actual code, not aspirational framework defaults.
